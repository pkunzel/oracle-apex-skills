# APEX_DATA_EXPORT

Source: [Oracle APEX 26.1 API Reference](https://docs.oracle.com/en/database/oracle/apex/26.1/aeapi/APEX_DATA_EXPORT.html)

Status: curated first-pass reference.

## Purpose

`APEX_DATA_EXPORT` turns an `APEX_EXEC.T_CONTEXT` into a downloadable file such as XLSX, PDF, HTML, CSV, XML, or JSON. Use it when PL/SQL owns the query context and needs to control columns, headings, formatting, aggregates, highlights, or print layout.

For a declarative region export, start with `APEX_REGION.EXPORT_DATA`. For an arbitrary SQL, REST Source, remote SQL, or custom `APEX_EXEC` context, start here.

## When To Use

Use `APEX_DATA_EXPORT` when:

- A page process needs to download data from a custom `APEX_EXEC` query.
- A plug-in or shared package needs a reusable export routine.
- You need column lists, headings, column groups, aggregates, or highlights.
- PDF output needs page size, orientation, header, footer, or styling control.
- Text output should be returned as a CLOB for downstream processing instead of immediately downloaded.

Avoid using it as a raw SQL execution helper. Open and close the query context with `APEX_EXEC`, then export that context.

## API Surface

| Need | Members |
| --- | --- |
| Export data | `EXPORT` |
| Send file to browser | `DOWNLOAD` |
| Select and format columns | `ADD_COLUMN`, `ADD_COLUMN_GROUP` |
| Summaries and emphasis | `ADD_AGGREGATE`, `ADD_HIGHLIGHT` |
| PDF/print layout | `GET_PRINT_CONFIG` |
| Constants and records | `Global Constants`, `Data Types` |

## Export And Download

Assuming a local table `emp(empno, ename, deptno)` and an APEX page process:

```sql
declare
    l_context apex_exec.t_context;
    l_export  apex_data_export.t_export;
begin
    l_context := apex_exec.open_query_context(
        p_location  => apex_exec.c_location_local_db,
        p_sql_query => q'[
            select empno, ename, deptno
              from emp
             order by ename
        ]');

    l_export := apex_data_export.export(
        p_context   => l_context,
        p_format    => apex_data_export.c_format_xlsx,
        p_file_name => 'employees');

    apex_exec.close(l_context);

    apex_data_export.download(
        p_export => l_export);
exception
    when others then
        apex_exec.close(l_context);
        raise;
end;
/
```

`DOWNLOAD` stops the APEX engine by default. Place it after validation, auditing, and context cleanup.

## Export Selected Columns To PDF

Assuming the same `emp` table:

```sql
declare
    l_context apex_exec.t_context;
    l_columns apex_data_export.t_columns;
    l_print   apex_data_export.t_print_config;
    l_export  apex_data_export.t_export;
begin
    apex_data_export.add_column(
        p_columns => l_columns,
        p_name    => 'EMPNO',
        p_heading => 'Employee No.');

    apex_data_export.add_column(
        p_columns     => l_columns,
        p_name        => 'ENAME',
        p_heading     => 'Name',
        p_is_frozen   => true);

    apex_data_export.add_column(
        p_columns => l_columns,
        p_name    => 'DEPTNO',
        p_heading => 'Department');

    l_print := apex_data_export.get_print_config(
        p_orientation => apex_data_export.c_orientation_landscape,
        p_page_header => 'Employee List',
        p_page_footer => 'Generated by Oracle APEX');

    l_context := apex_exec.open_query_context(
        p_location  => apex_exec.c_location_local_db,
        p_sql_query => 'select empno, ename, deptno from emp order by deptno, ename');

    l_export := apex_data_export.export(
        p_context      => l_context,
        p_format       => apex_data_export.c_format_pdf,
        p_columns      => l_columns,
        p_print_config => l_print,
        p_file_name    => 'employees-by-department');

    apex_exec.close(l_context);
    apex_data_export.download(l_export);
exception
    when others then
        apex_exec.close(l_context);
        raise;
end;
/
```

## Return Text Output For Processing

Use `p_as_clob => true` only for text formats such as CSV, XML, JSON, or HTML:

```sql
declare
    l_context apex_exec.t_context;
    l_export  apex_data_export.t_export;
begin
    l_context := apex_exec.open_query_context(
        p_location  => apex_exec.c_location_local_db,
        p_sql_query => 'select empno, ename from emp');

    l_export := apex_data_export.export(
        p_context   => l_context,
        p_format    => apex_data_export.c_format_json,
        p_as_clob   => true,
        p_file_name => 'employees');

    apex_exec.close(l_context);

    -- Assuming an audit table with a CLOB column.
    insert into export_audit(file_name, payload_clob, created_on)
    values ('employees.json', l_export.content_clob, systimestamp);
end;
/
```

## Notes

- Always close the `APEX_EXEC` context after `EXPORT` has consumed it.
- Use `APEX_REGION.EXPORT_DATA` when you want the current state of a declarative region, including the current saved report/view.
- Use `p_file_name` without an extension when `DOWNLOAD` uses `p_add_file_extension => true`.
- `p_as_clob` does not apply to binary formats such as XLSX and PDF.
- Treat filters and SQL fragments as untrusted if they come from users. Prefer bind parameters through `APEX_EXEC`.
